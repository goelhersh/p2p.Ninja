A brief review of how you may edit the settings in your gateway:

VERSION, Ethernet IP and TCP_PORT can only be changed by you directly editing the settings file of the p2p.Ninja installation folder. You may do this by removing the SD card from the raspberry pi and browsing to the p2p.Ninja installation folder. This is best done in a computer that can read linux formated disks. Alternatively, if you have a monitor and keyboard attached to the raspberry pi, you should be able to use the Raspberry pi itself to log in and edit p2p.Ninja/Usersettingsandparameters/settings. This method also allows you to edit settings that are discussed here.

parameter_to_be_changed Details Additional notes
SELF_PORT_PEER This is the ip address of your p2p.Ninja gateway that is visible to it's peers on the p2p.Ninja network. It is recommended that you change this settting to a Unique identifier after setting up your p2p.Ninja gateway for the first time.
LOCALID A unique identifier of your installation/node of the p2p.Ninja gateway. It is recommended that you change this settting to a Unique identifier after setting up your p2p.Ninja gateway for the first time.
LOCALROUTINGAID optional parameter that identifies the locality or 'local network' that this p2p.Ninja gateway is part of.
LOCALIDPUBLICKEY if 'True', indicates that LOCALID value is a RSA encryption public key
LOCALKEYEXPONENT only needed if LOCALIDPUBLICKEY is true.
ACCEPTABLEHOPCOUNT Indicates the maximum number of hops that that a packet may have. IF the packet has more hops that this number, the packet is discarded by the router.
NUMBEROFKEYSTOSAVE Indicates how many destination addresses are saved in the routing tables before older addresses (and their associated routing details) are deleted.
MINIMUMINTERVALBETWEENBROADCASTS If another broadcast message is encountered from a node before this duration has elapsed, it will be ignored.
NUMBEROFCONNECTIONSTOSAVE Indicates the maximum number of client connections (typically on port 80) that can be simultaneously open. If a new connection request is recieved, the oldest connection will be closed.

Please ensure you give IP address in the right format ( for example 192.168.3.4 ). LOCALROUTINGAID and PASSWORD may be alfanumeric, other values must be integers. Please do not input spaces into the fields provided.
